## Overview

This workflow automates the creation of staging environments that mirror production, using AI to generate configuration files and CI/CD pipelines to sync them. By leveraging Env0's AI-powered environment generation and GitHub Actions for automation, you can spin up staging environments on demand and keep them synchronized with production data and configuration.

## Prerequisites

- A cloud provider account (AWS, GCP, or Azure)
- GitHub repository with your infrastructure-as-code (Terraform)
- Env0 account (free tier available)
- Basic familiarity with Terraform and CI/CD

## Step 1: Set Up Env0 for AI Configuration Generation

1. Sign up at [env0.com](https://www.env0.com) and create a new project.
2. Connect your cloud provider (e.g., AWS) by adding credentials in Env0.
3. Use Env0's AI assistant to generate Terraform configurations for your staging environment:
   - Navigate to **Templates** → **New Template** → **Generate with AI**.
   - Describe your infrastructure (e.g., "VPC with two subnets, an RDS database, and an EC2 instance running nginx").
   - Env0 will output a Terraform configuration. Review and save it as a template.

## Step 2: Create a GitHub Actions Workflow for Staging Environment Creation

1. In your GitHub repository, create `.github/workflows/create-staging.yml`.
2. Add the following workflow to trigger on demand (workflow_dispatch) or on pull request:

```yaml
name: Create Staging Environment
on:
  workflow_dispatch:
    inputs:
      environment_name:
        description: 'Staging environment name'
        required: true
        default: 'staging-001'

jobs:
  create:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v2
        with:
          terraform_version: 1.5.0
      - name: Configure Env0 CLI
        run: |
          curl -sSL https://get.env0.com | sh
          env0 login --api-key ${{ secrets.ENV0_API_KEY }}
      - name: Deploy Staging Environment
        run: |
          env0 deploy --template staging-template \
            --environment ${{ github.event.inputs.environment_name }} \
            --variables "db_password=${{ secrets.DB_PASSWORD }}"
```

3. Add your Env0 API key as a GitHub secret (`ENV0_API_KEY`).

## Step 3: Sync Staging with Production Data

1. Use a scheduled GitHub Actions workflow to sync production database to staging:

```yaml
name: Sync Production to Staging
on:
  schedule:
    - cron: '0 3 * * *'  # daily at 3 AM
  workflow_dispatch:

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Dump production DB
        run: |
          pg_dump ${{ secrets.PROD_DATABASE_URL }} > prod_dump.sql
      - name: Restore to staging DB
        run: |
          psql ${{ secrets.STAGING_DATABASE_URL }} < prod_dump.sql
```

2. For file storage (e.g., S3), use `aws s3 sync` to copy production bucket to staging bucket.

## Step 4: Automate Configuration Sync with AI

1. Set up a webhook in Env0 to trigger when production configuration changes (e.g., via Terraform Cloud webhook).
2. Use Env0's AI to compare production and staging configurations and generate a diff patch.
3. Apply the patch automatically using a GitHub Actions workflow that runs on the webhook.

## Step 5: Teardown Staging Environments

To avoid costs, automatically destroy staging environments after a set time:

```yaml
name: Destroy Staging Environment
on:
  workflow_dispatch:
    inputs:
      environment_name:
        description: 'Staging environment name'
        required: true

jobs:
  destroy:
    runs-on: ubuntu-latest
    steps:
      - name: Destroy environment
        run: |
          env0 destroy --environment ${{ github.event.inputs.environment_name }}
```

## Conclusion

By combining Env0's AI configuration generation with GitHub Actions, you can automate the entire lifecycle of staging environments—creation, sync, and teardown—saving hours of manual work and reducing errors. This setup ensures your staging environment is always a reliable mirror of production.
